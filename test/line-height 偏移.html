android 移动端通过line-height来实现文字上下居中，视觉上不是居中(line-height比较小且为单数像素值特别明显)；可以通过补上 1-2px 的 padding-top。

上下居中 line-height: 24px;

上下居中 line-height: 25px;

上下居中 line-height: 24px; padding: 1px 0 0;

上下居中 line-height: 24px; padding: 2px 0 0;

上下居中 line-height: 25px; padding: 1px 0 0;

上下居中 line-height: 25px; padding: 2px 0 0;

如果line-height为双数像素值，补1px较好；如果line-height为单数像素值，2px较好。
要注意，如果没固定高度的情况下，单行文字line-height的值，就是content的高度，所以最终元素的高度还要算上padding，border；所以如果要按照设置图做指定高度时，设置padding-top后，要减少一下line-height。

要求24px line-height: 23px; padding: 1px 0 0;

要求24px line-height: 22px; padding: 2px 0 0;

要求25px line-height: 24px; padding: 1px 0 0;

要求25px line-height: 23px; padding: 2px 0 0;

----line-height:25px----

感觉line-height的值为单数像素值的时候，元素就是直接在双数值的下面加了1px。内容并没有下移，这也是为什么line-height为单数像素值视觉不居中特别明显。不过也正常，css的像素一般是整数才有效的，小数会自动四舍五入。
除了补padding-top以外，还可以固定高度，line-height设得比高度大1、2px
